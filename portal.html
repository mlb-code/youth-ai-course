למה באתם לקורס? מה אתם רוצים ללמוד? האם באתם להכיר את הכלים ולהנות, או שיש לכם מטרה ספציפית? • מה כבר שמעתם על בינה מלאכותית?

  • מה זה AI? בינה מלאכותית היא כשמחשבים לומדים לבצע משימות שדורשות חשיבה אנושית — הבנת שפה, זיהוי תמונות, קבלת החלטות. בניגוד לתוכנות רגילות שעובדות לפי כללים קשיחים, AI לומד מנתונים
  • סוגי AI: טקסט (ChatGPT, Claude), תמונות (DALL-E, NanoBanana), וידאו (Sora, Novo), קוד (Cursor), מוזיקה (Suno). כל סוג מעבד סוג אחר של מידע
  • AI בחיים היומיום: Siri ו-Google Assistant (קול), נטפליקס (המלצות), Google Translate (שפה), Waze (ניווט), Face ID (זיהוי), פילטרי ספאם (אימייל)
  • הכרות עם ChatGPT ו-Claude: שני הכלים החזקים ביותר. נכנסים, שואלים את אותה שאלה בשניהם, ורואים את ההבדלים — כל כלי חזק בדברים שונים
  • היסטוריה קצרה: הרעיון של AI קיים מ-1950 (אלן טיורינג), אבל הפריצה הגדולה הגיעה ב-2022 עם ChatGPT. ישראל מובילה עולמית בתחום!

AI
מטרה: ללמוד לדבר עם AI בצורה אפקטיבית + התקנת Cursor ו-Claude Code

  • מה זה פרומפט? ההוראה או השאלה שנותנים ל-AI. חשבו על זה כמו לדבר עם עוזר חכם מאוד — ככל שתהיו ברורים יותר, כך התוצאה תהיה טובה יותר
  • טכניקה 1 — היו ספציפיים: במקום "כתוב על כלבים" כתבו "כתוב פוסט של 200 מילים על 5 גזעי כלבים מומלצים למשפחות עם ילדים קטנים"
  • טכניקה 2 — תנו הקשר: "אני תלמיד בן 15 שבונה סטארטאפ. עזור לי לכתוב..." — ה-AI צריך לדעת מי אתם ולמה אתם צריכים את זה
  • טכניקה 3 — הגדירו פורמט: "תן לי רשימה ממוספרת / טבלה / 3 פסקאות / נקודות" — תגידו ל-AI בדיוק איך אתם רוצים את התשובה
  • טכניקה 4 — תנו דוגמה: "כתוב בסגנון דומה ל-X, אבל עם Y" — דוגמאות עוזרות ל-AI להבין מה אתם מחפשים
  • טכניקה 5 — שפרו צעד אחר צעד: התחילו פשוט, ואז בקשו שינויים: "תקצר", "תוסיף הומור", "עכשיו תרגם לעברית". כל פעם מקבלים תוצאה טובה יותר!
  • 💻 התקנת Cursor + Claude Code: בחצי השני של המפגש — מתקינים את Cursor (עורך הקוד עם AI) ו-Claude Code (הכלי החכם ביותר לבנייה). ההדרכה המלאה באזור "הדרכות"

חשיבה יזמית — מוצאים בעיות

יזמות
מטרה: ללמוד לחשוב כמו יזמים, לזהות בעיות אמיתיות ולהבין את הנוסחה של סטארטאפ מצליח
🔙 במפגש הקודם

למדנו לכתוב פרומפטים מושלמים והתקנו Cursor ו-Claude Code. • בדיקת משימת בית: מי הצליח ליצור לוגו עם AI? מי נרשם ל-Claude Code? הראו את התוצאות!

  • מה זה סטארטאפ? חברה צעירה שפותרת בעיה אמיתית בדרך חדשה או טובה יותר. ההבדל מעסק רגיל: סטארטאפ שואף לצמוח מהר ולהגיע להרבה אנשים
  • סיפורי יזמים ישראליים: Waze (פתרו את בעיית הפקקים), Wix (הקלו על בניית אתרים), Monday (סידרו כאוס בניהול פרויקטים), Iron Dome (ביטחון). ישראל = "אומת הסטארטאפ" — יותר סטארטאפים לנפש מכל מדינה!
  • הנוסחה: בעיה ← פתרון ← מוצר ← עסק. הכל מתחיל מבעיה אמיתית שאנשים רוצים לפתור!
  • Design Thinking: שיטה בת 5 שלבים — הזדהו (הבינו את המשתמש), הגדירו (מה הבעיה בדיוק), חשבו (סיעור מוחות), בנו (אב-טיפוס מהיר), בדקו (קבלו פידבק)
  • איך מוצאים בעיות טובות? מה מעצבן אתכם ביומיום? על מה חברים מתלוננים? מה הייתם רוצים שיהיה קל יותר? הבעיות הכי טובות הן אלה שאתם חווים בעצמכם!
  • בעיה טובה vs בעיה רעה: טובה = הרבה אנשים חווים אותה, מוכנים לשלם על פתרון, ספציפית. רעה = עמומה מדי, רק אתם חווים אותה, כבר נפתרה לגמרי
סטארטאפ Design Thinking אימות בעיה בעיה ← פתרון

מרעיון למוצר

אסטרטגיה
מטרה: לבחור את הרעיון הטוב ביותר, להגדיר קהל יעד, לבנות הצעת ערך ולתכנן MVP
🔙 במפגש הקודם

למדנו חשיבה יזמית ומצאנו 3 בעיות מהחיים שלנו. • בדיקת משימת בית: מי בחר את הבעיה הטובה ביותר? מי שאל 5 אנשים? מה המתחרים שמצאתם?

  • בחירת הרעיון: מתוך 3 הבעיות מהמפגש הקודם, בוחרים את הטובה ביותר. הקריטריונים: האם הבעיה אמיתית? האם אני מבין אותה? האם אפשר לבנות פתרון? האם אנשים ירצו להשתמש?
  • קהל יעד: מי בדיוק סובל מהבעיה? לא "כולם" אלא "תלמידי תיכון בישראל שמתקשים במתמטיקה". בונים פרסונה — דמות בדיונית שמייצגת את המשתמש האידיאלי
  • הצעת ערך: מה מיוחד בפתרון שלכם? הנוסחה: "אנחנו עוזרים ל-[קהל] לפתור את [הבעיה] על ידי [הפתרון], בניגוד ל-[מתחרים] אנחנו [יתרון ייחודי]"
  • ניתוח מתחרים: מה כבר קיים? חפשו בגוגל, הורידו אפליקציות דומות. מה טוב בהן? מה חסר? ההזדמנות שלכם נמצאת בפערים!
  • MVP: Minimum Viable Product — הגרסה הכי פשוטה שעדיין שימושית. לא המוצר המלא, רק מספיק כדי לבדוק אם אנשים רוצים את זה

בונים אתר — חלק 1

בנייה
מטרה: התקנת Cursor, הבנת מבנה בסיסי של אתר, ובנייה ראשונה של דף נחיתה עם AI
🔙 במפגש הקודם

בחרנו רעיון, הגדרנו קהל יעד ומילאנו את "בונה הפרויקט". • בדיקת משימת בית: מי השלים את 10 השאלות? מי חשב על עיצוב וצבעים? הציגו את האפיון שלכם!

  • מה זה Cursor? Cursor הוא עורך קוד חכם (מבוסס על VS Code) עם בינה מלאכותית מובנית. במקום ללמוד שפות תכנות — אתם פשוט מתארים מה אתם רוצים לבנות בשפה חופשית, ו-Cursor כותב את כל הקוד בשבילכם! הוא יודע לבנות אתרים שלמים, לתקן באגים, להוסיף עיצוב ואנימציות — הכל לפי מה שתבקשו. לא צריך ידע קודם בתכנות!
  • התקנה צעד אחר צעד: (1) היכנסו ל-cursor.com ולחצו Download. (2) הריצו את קובץ ההתקנה (EXE בווינדוס, DMG במק). (3) פתחו את Cursor והירשמו עם חשבון Google — הגרסה החינמית מספיקה! (4) לחצו File → Open Folder וצרו תיקייה חדשה בשם הפרויקט (באנגלית, בלי רווחים! למשל: my-startup). (5) לחצו Ctrl+I (ווינדוס) או Cmd+I (מק) לפתיחת חלון ה-AI — וכתבו מה אתם רוצים לבנות!
  • מבנה אתר — 3 שכבות: כל אתר באינטרנט בנוי מ-3 שפות שעובדות ביחד: HTML = השלד — מגדיר מה יש בדף (כותרות, טקסטים, תמונות, כפתורים). CSS = העיצוב — קובע איך הדף נראה (צבעים, גדלים, פונטים, מרווחים, פריסה). JavaScript = המוח — מפעיל דברים (מה קורה כשלוחצים כפתור, אנימציות, טפסים). הדפדפן (Chrome) קורא את 3 הקבצים האלה ומציג את הדף על המסך.
  • מה זה שרת? איך אתר עובד? כשאתם כותבים כתובת אתר בדפדפן (למשל google.com), המחשב שלכם (קליינט) שולח בקשה לשרת — מחשב חזק שיושב באיזה מרכז נתונים בעולם ומחזיק את קבצי האתר. השרת שולח חזרה את קבצי ה-HTML/CSS/JS והדפדפן שלכם מציג אותם. דומיין (כמו google.com) = הכתובת הידידותית. אחסון (Hosting) = השרת שמאחסן את הקבצים. כרגע אנחנו בונים מקומית (הקובץ רק על המחשב שלכם) — במפגש הבא נלמד להעלות את האתר לאינטרנט!
  • בנייה עם AI — הפרומפט הראשון: לוחצים Ctrl+I (ווינדוס) או Cmd+I (מק), וכותבים פרומפט מפורט באנגלית (ה-AI מבין אנגלית טוב יותר, אבל התוצאה תהיה בעברית!). לדוגמה: "Build me a landing page for [שם המוצר] with a hero section with catchy headline, about section, features with 3 cards and icons, testimonials, and a contact form. Use [צבעים] colors, Hebrew RTL, modern design with gradients." ככל שהפרומפט מפורט יותר — התוצאה טובה יותר!
  • תצוגה מקדימה: כדי לראות את האתר שבניתם, מחפשים את קובץ ה-index.html בתיקייה ופותחים אותו בדפדפן Chrome (לחיצה כפולה על הקובץ). אחרי כל שינוי ב-Cursor — חוזרים לדפדפן ולוחצים רענן (F5) כדי לראות את העדכון. זה הזמן לבדוק: האם הטקסט קריא? האם הצבעים נכונים? האם הכפתורים במקום?

היכנסו ל-github.com ולחצו "Sign up". בחרו שם משתמש (באנגלית, בלי רווחים), הזינו אימייל וסיסמה. אשרו את האימייל שנשלח.

3

התקנת Git

Git זו תוכנה שרצה על המחשב ומתחברת ל-GitHub. הורידו מ-git-scm.com והתקינו עם הגדרות ברירת מחדל (Next → Next → Install).

4

חיבור ל-Cursor

Cursor כבר יודע לעבוד עם Git! פתחו Terminal ב-Cursor (Ctrl+`) והגדירו את השם והאימייל שלכם:

git config --global user.name "Your Name"
git config --global user.email "your@email.com"

העלאת פרויקט

ב-GitHub לחצו "New repository", תנו שם לפרויקט, ולחצו Create. העתיקו את הפקודות שיופיעו והדביקו ב-Terminal של Cursor. הפרויקט שלכם עכשיו באוויר!
